# 3.5.3 Benchmark Harness Hooks

## Design

- The compiler emits benchmarkable standalone C, not only kernels.
- The harness code serves three purposes:
  load input data
  verify correctness
  measure performance and collect matrix features.
- This is why code generation is split between kernel emission and runtime emitters.

## Implementation

- `runtime_emitter.cpp` emits:
  headers and includes
  `SparseMatrix` definition
  Matrix Market loader
  COO-to-CSR and COO-to-CSC conversion
  timing helpers
  feature extraction
  `main()`.
- `reference_emitter.cpp` emits verification kernels.
- Sparse-output support emits:
  assembly helpers for output structure
  sparse-value comparison helpers
  optional hash-map support for dynamic sparse accumulation.

## Harness Features

- Matrix Market loader:
  reads coordinate format
  handles real/integer/pattern fields
  expands symmetric and skew-symmetric inputs
  canonicalizes ordering
  merges duplicate coordinates.
- Timing utilities:
  `get_time_ms()`
  average/min/max/stddev reporting logic in emitted `main()`.
- Feature extraction:
  density
  row and column irregularity
  bandwidth and row span
  clustering/gap measures
  normalized entropy
  cross-row locality proxy.

## CLI Integration

- `src/main.cpp` chooses output mode and forwards optimization settings to codegen.
- Emitted files include optimization comments and, when applicable, schedule-order comments.
- The generated program is meant to be compiled with `gcc -O2` and run against Matrix Market inputs.
