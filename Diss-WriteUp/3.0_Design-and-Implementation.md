# 3 Design and Implementation

## Summary

- This section documents the implemented compiler pipeline, not an idealized architecture.
- The active path in this repository is:
  DSL source -> Flex/Bison parse -> AST -> semantic IR -> scheduled IR -> scheduled optimizations -> C code generation.
- The implementation supports two compilation modes:
  single-compute lowering for one `compute` statement
  scheduled-program lowering for multi-statement inputs, `call`, and `for` regions.
- Existing topical notes in this folder remain useful as source material, but the numbered files are the working set for Section 3.

## Design

- The core design split is semantic meaning vs executable schedule.
- The DSL captures tensor algebra intent and sparse formats.
- Semantic IR classifies tensors, indices, and output strategy.
- Scheduled IR materializes the loop tree, sparse iterators, merge behavior, block wrappers, and loop-local statements.
- Optimizations mutate scheduled IR rather than source syntax or emitted C.
- Code generation is split between:
  compute-kernel emission from scheduled IR
  non-kernel emitters for loaders, reference kernels, sparse output assembly, and benchmark harness code.

## Implementation

- CLI entrypoint: `src/main.cpp`.
- Parser and lexer: `src/parser.y`, `src/lexer.l`.
- AST definitions: `include/ast.h`.
- Shared IR and structured statement/expression nodes: `include/ir.h`, `src/ir.cpp`.
- Semantic and scheduled lowering: `include/semantic_ir.h`, `src/semantic_ir.cpp`.
- Optimization passes: `include/scheduled_optimizations.h`, `src/scheduled_optimizations.cpp`.
- Code generation entrypoints: `include/codegen.h`, `src/codegen.cpp`, `src/codegen/*.cpp`.
- Test evidence is distributed across:
  parser tests in `src/tests/parser/*`
  IR tests in `src/tests/ir/*`
  optimization tests in `src/tests/optimizations/*`
  codegen tests in `src/tests/codegen/*`.
