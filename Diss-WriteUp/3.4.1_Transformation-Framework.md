# 3.4.1 Transformation Framework

## Design

- The framework is a small pass manager specialized for scheduled sparse loop trees.
- Its job is not to search a large schedule space; it deterministically applies enabled transformations in a chosen order.
- The main abstraction boundary is:
  `OptConfig` describes what to do
  the scheduled optimization pass file performs the mutations
  codegen reads the resulting metadata and structure.

## Implementation

- `OptConfig` fields:
  `enableBlocking`
  `enableInterchange`
  `blockSize`
  `enable2DBlocking`
  `blockSize2`
  `order`
  `outputFile`.
- Convenience constructors:
  `baseline()`
  `blockingOnly(...)`
  `interchangeOnly()`
  `allOptimizations(...)`
  `blocking2D(...)`.
- Program dispatch is recursive:
  `applyOptimizations(Program&, ...)` walks statements
  `applyOptimizationsToStmt(...)` descends into `Region` bodies
  compute statements are transformed individually.

## Framework Mechanics

- Blocking locates a target dense loop by index name and wraps it with a block loop.
- Interchange walks the tree depth-first and tries the first legal profitable local swap pattern it recognizes.
- Metadata is recorded into `ir::LoopOptimizations` rather than inferred again later.

## Design Consequence

- The framework is simple enough to test with golden schedules and codegen snapshots.
- It is also intentionally narrow:
  there is no pass registration API
  there is no general dependence solver
  there is no profile-guided schedule search in the compiler itself.
