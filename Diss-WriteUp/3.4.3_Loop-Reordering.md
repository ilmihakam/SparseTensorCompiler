# 3.4.3 Loop Reordering

## Design

- The repository contains two distinct reorder-style transformations:
  format-correctness reordering
  loop interchange.
- They should not be conflated.
- Format-correctness reordering restores traversal order that matches sparse storage.
- Loop interchange is a performance transformation that changes nesting to improve locality or accumulation behavior.

## Implementation

- `applyReordering(...)`:
  identifies the primary sparse tensor
  compares current loop order with a computed natural order
  swaps the root and child headers when the two-deep order is reversed.
- `computeNaturalOrder(...)` uses tensor format plus tensor index names to decide the preferred outer/inner order.
- `applyLoopInterchange(...)` performs a depth-first search and delegates to `tryInterchangeAtDenseNode(...)`.

## Interchange Mechanics

- The pass looks for dense-rooted local patterns and rewrites subtree structure.
- It handles:
  dense -> sparse -> dense
  dense -> dense -> sparse
- Helper routines fuse accumulator patterns when moving a dense loop outside a sparse loop so the resulting scheduled body remains valid.

## Practical Meaning

- Interchange can move a dense loop outside a sparse loop in SpMM-style schedules so accumulation behavior changes from repeated updates to a more locality-friendly organization.

## Metadata

- Reordering records original and new orders.
- Interchange records a boolean `interchangeApplied`.
- Codegen uses this metadata for emitted comments; it does not rerun legality logic.
