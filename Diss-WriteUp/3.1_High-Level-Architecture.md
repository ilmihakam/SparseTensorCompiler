# 3.1 High-Level Architecture

## Design

- The compiler is organized as a staged lowering pipeline rather than a direct source-to-C translator.
- Each stage answers a narrower question:
  AST: what was written
  semantic IR: what the tensor equation means
  scheduled IR: how to traverse the iteration space
  codegen: how to emit runnable C.
- Sparse formats are treated as first-class metadata early in the pipeline so later stages can make storage-aware scheduling decisions.
- Optimizations are attached after scheduling because legality depends on loop structure, sparse dependencies, and runtime bounds.

## Implementation

- `src/main.cpp` is the driver:
  reads the `.tc` file
  parses into `g_program`
  detects whether the input needs program mode
  lowers and optimizes
  emits C to the requested output file.
- Program-path trigger conditions in `compileFile(...)`:
  presence of `ForStatement`
  presence of `CallStatement`
  more than one `Computation`.
- Single-compute path:
  `lowerFirstComputationToScheduledOptimized(...)`
  returns one `scheduled::Compute`
  applies reordering/interchange/blocking before code generation.
- Program path:
  `lowerToSemanticProgram(...)`
  `scheduleProgram(...)`
  `opt::applyOptimizations(scheduled::Program&, ...)`
  `codegen::generateProgramToFile(...)`.
- Code generation is split across files:
  `src/codegen.cpp`: scheduled loop and structured IR emission
  `src/codegen/runtime_emitter.cpp`: full standalone C wrapper
  `src/codegen/reference_emitter.cpp`: verification/reference kernels
  `src/codegen/output_assembly_emitter.cpp` and `output_assembly_shared.cpp`: sparse output construction
  `src/codegen/program_emitter.cpp`: scheduled-program mode.

## Main Data Flow

- Parser builds AST nodes declared in `include/ast.h`.
- Semantic lowering converts AST expressions to `sparseir::Expr` and classifies indices, inputs, and output strategy.
- Scheduling converts semantic iterator structure into `scheduled::Loop` trees with explicit header kinds:
  `DenseFor`
  `SparseIterator`
  `SparseMerge`
  `Block`.
- Optimization passes mutate the scheduled tree and set `ir::LoopOptimizations`.
- `CodeGenerator` consumes the scheduled compute plus optimization metadata and emits C.

## Constraints Visible in the Current Code

- Active format lowering only distinguishes `Dense`, `CSR`, and `CSC`.
- Parallel execution is not modeled.
- Sparse-output kernels use extra assembly/reference helpers because output structure may need to be materialized separately from compute.

## Primary Code Paths

- `src/main.cpp`
- `src/semantic_ir.cpp`
- `src/scheduled_optimizations.cpp`
- `src/codegen.cpp`
- `src/codegen/runtime_emitter.cpp`
