# 3.2 Domain-Specific Language (DSL)

## Design

- The DSL is a small tensor-algebra language for expressing sparse linear algebra kernels in index notation.
- It separates algorithm intent from loop schedule:
  the source states tensor accesses and index relationships
  the compiler chooses sparse iterators, merge structure, and loop transformations.
- The language is format-annotated rather than format-inferred.
- This is important for the implementation because CSR and CSC induce different sparse axes and therefore different loop headers.
- The supported statement classes are intentionally small:
  tensor declarations
  `compute` assignments
  `call` statements
  `for` regions.
- The DSL supports operations such as addition, matrix multiplication, vector multiplication, element wise multiplication, etc which allows them to support kernels like SpMV, SpMM, SpAdd, SpElMul, SDDMM, and SpGEMM

## Implementation

- Parsing is defined by Bison in `src/parser.y`.
- Lexical rules are defined by Flex in `src/lexer.l`.
- AST nodes are implemented in `include/ast.h`.
- `Declaration` stores tensor name, source-level format string, and optional shape list.
- `Computation` stores a tensor access on the left-hand side and an expression tree on the right-hand side.
- `CallStatement` and `ForStatement` are parsed and represented explicitly; they are not just placeholders in the grammar.
- Format mapping happens during semantic lowering:
  `stringToFormat(...)` maps `CSR` and `CSC` explicitly
  any other accepted format keyword currently falls back to `ir::Format::Dense`.

## What the DSL Expresses Well

- Dense and sparse tensor declarations with optional static dimensions.
- Einstein-style computations where repeated RHS-only indices become reductions.
- Mixed sparse-dense contractions such as `compute C[i, j] = A[i, k] * B[k, j];`.
- Pointwise sparse operations where output pattern is derived from input sparsity.

## What the language does not support

- No direct schedule syntax in the language itself.
- No explicit reduction operator; reductions are inferred from index usage.
- No parallel, SIMD, or target-specific directives.

- `src/parser.y`
- `src/lexer.l`
- `include/ast.h`
