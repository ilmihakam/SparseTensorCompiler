# 3.6 Code Walkthrough: CSR SpMM

## 1. Example DSL

- The walkthrough uses the canonical CSR SpMM example already exercised throughout the repository.

```tc
tensor C : Dense<100, 50>;
tensor A : CSR<100, 80>;
tensor B : Dense<80, 50>;
compute C[i, j] = A[i, k] * B[k, j];
```

- Interpretation:
  `A` is sparse in CSR form
  `B` is dense
  `C` is dense output
  free indices are `i`, `j`
  reduction index is `k`.

## 2. IR Showing the Exact Computation

- At the semantic level, the computation is:

```text
Compute(C[i, j] = A[i, k] * B[k, j])
```

- Semantic facts derived by `lowerComputationToSemantic(...)`:
  output = `C : Dense<100, 50> [i, j]`
  inputs = `A : CSR<100, 80> [i, k]`, `B : Dense<80, 50> [k, j]`
  free indices = `[i, j]`
  reduction indices = `[k]`
  `outputStrategy = DenseArray`.

- The scheduled IR snapshot used by the repository for the baseline case is:

```text
Output:
  Tensor C : Dense<100, 50> [i, j]

Inputs:
  Tensor A : CSR<100, 80> [i, k]
  Tensor B : Dense<80, 50> [k, j]

Loop Nest:
  for i in [0, 100) (dense):
    for k in [0, 80) (sparse):
      for j in [0, 50) (dense):
        C[i][j] += A_vals[pA] * B[k][j];
```

- Meaning of this loop tree:
  `i` is the outer dense row loop
  `k` is driven by CSR traversal of row `i`
  `j` is the inner dense output-column loop
  the sparse loop binds `k` from `A->col_idx[pA]`.

## 3. IR Transformations

### 3.a IR Showing the Loop Interchange Transformation

- The interchange pass swaps the dense `j` loop outward across the sparse `k` loop when the local pattern is legal.
- Repository snapshot for the CSR interchange case:

```text
Loop Nest:
  for i in [0, 100) (dense):
    for j in [0, 50) (dense):
      for k in [0, 80) (sparse):
        C[i][j] += A_vals[pA] * B[k][j];

Optimizations:
  Format-correctness reordering: not needed
  Interchange: applied
  Blocking: not applied
```

- Structural change:
  baseline = `i -> k -> j`
  interchange = `i -> j -> k`.

### 3.b IR Showing the 2D Code Blocking Transformation

- The 2D blocking path wraps both dense dimensions with block loops.
- For CSR SpMM, the implemented 2D blocking shape is:

```text
Loop Nest:
  for i_block in [0, ceil(100 / 32)) (block):
    for i in [i_start, i_end) (dense):
      for k in [0, 80) (sparse):
        for j_block in [0, ceil(50 / 64)) (block):
          for j in [j_start, j_end) (dense):
            C[i][j] += A_vals[pA] * B[k][j];

Optimizations:
  Format-correctness reordering: not needed
  Interchange: not applied
  Blocking: 2D, indices=(i, j), block_sizes=(32, 64)
```

- This matches the implemented 2D blocking structure checked by the tests:
  `i_block -> i -> k(sparse) -> j_block -> j`.

- Structural change:
  baseline = `i -> k -> j`
  2D blocking = `i_block -> i -> k -> j_block -> j`.

## 4. Codegen

### 4.a Codegen Showing the Baseline Kernel

- Repository artifact: `artifacts/baseline/spmm_csr_baseline.kernel.c.txt`.

```c
void compute(const SparseMatrix* A, const double** B, double** C, int N_j) {
    for (int i = 0; i < A->rows; i++) {
        for (int pA = A->row_ptr[i]; pA < A->row_ptr[i + 1]; pA++) {
            int k = A->col_idx[pA];
            for (int j = 0; j < N_j; j++) {
                C[i][j] += A->vals[pA] * B[k][j];
            }
        }
    }
}
```

- This is the direct rendering of the baseline scheduled loop tree.

### 4.b Codegen Showing Loop Interchange

- Repository artifact: `artifacts/interchange/spmm_csr_interchange.kernel.c.txt`.

```c
void compute(const SparseMatrix* A, const double** B, double** C, int N_j) {
    // Optimization: loop interchange applied
    for (int i = 0; i < A->rows; i++) {
        for (int j = 0; j < N_j; j++) {
            for (int pA = A->row_ptr[i]; pA < A->row_ptr[i + 1]; pA++) {
                int k = A->col_idx[pA];
                C[i][j] += A->vals[pA] * B[k][j];
            }
        }
    }
}
```

- Visible effect:
  the dense output-column loop `j` is now outside the sparse iterator loop.

### 4.c Codegen Showing 2D Code Blocking

- Generated from the same DSL with `--opt-block-2d=32x64`.

```c
void compute(const SparseMatrix* A, const double** B, double** C, int N_j) {
    // Optimization: loop blocking (block_size=32)
    for (int i_block = 0; i_block < (A->rows + 31) / 32; i_block++) {
        int i_start = i_block * 32;
        int i_end = (i_start + 32 < A->rows) ? i_start + 32 : A->rows;
        for (int i = i_start; i < i_end; i++) {
            for (int pA = A->row_ptr[i]; pA < A->row_ptr[i + 1]; pA++) {
                int k = A->col_idx[pA];
                for (int j_block = 0; j_block < (N_j + 63) / 64; j_block++) {
                    int j_start = j_block * 64;
                    int j_end = (j_start + 64 < N_j) ? j_start + 64 : N_j;
                    for (int j = j_start; j < j_end; j++) {
                        C[i][j] += A->vals[pA] * B[k][j];
                    }
                }
            }
        }
    }
}
```

- Visible effect:
  the row dimension is tiled by `i_block`
  the column dimension is tiled by `j_block`
  both logical dense dimensions are bounded by derived start/end variables.

## Notes

- For this CSR SpMM case, format-correctness reordering is not needed because the source-level access order already matches CSR traversal.
- The walkthrough above intentionally uses repository artifacts rather than hand-written pseudocode so that the dissertation notes match the tested implementation.
