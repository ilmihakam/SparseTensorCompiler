# 3.5 Code Generation

## Design

- Code generation is split by responsibility rather than concentrated in one monolithic emitter.
- The central design rule is:
  scheduled compute structure is emitted from scheduled IR
  surrounding runtime/harness logic is emitted by dedicated helper modules.
- This keeps the kernel path close to the scheduled loop tree and reduces duplicated schedule logic.

## Implementation

- Public API in `include/codegen.h` exposes:
  `generateCode(...)`
  `generateKernelCode(...)`
  `generateToFile(...)`
  `generateKernelToFile(...)`
  `generateProgramToFile(...)`.
- `CodeGenerator` owns the main backend state:
  active compute
  active optimization config
  `EmissionContext`
  indentation/output helpers.
- `EmissionContext` summarizes:
  sparse and dense inputs
  output tensor
  output strategy/pattern
  primary sparse format
  loop-derived runtime bounds
  explicit dimension parameters
  nested sparse traversal status.

## Backend Split

- `src/codegen.cpp`:
  scheduled loop emission
  structured IR expression and statement emission
  kernel signatures
  kernel body construction.
- `src/codegen/runtime_emitter.cpp`:
  full standalone C structure
  matrix loading/conversion
  timing
  feature extraction
  `main()`.
- `src/codegen/reference_emitter.cpp`:
  reference implementations for dense and sparse-output cases.
- `src/codegen/output_assembly_emitter.cpp` and `output_assembly_shared.cpp`:
  helper code that constructs sparse output structure for union/intersection/sampled/dynamic-row modes.
- `src/codegen/program_emitter.cpp`:
  scheduled-program mode beyond the single-compute API.

## Current Scope

- Generated output targets portable CPU C.
- The backend emits benchmark-ready programs, not only raw kernels.
- Sparse outputs may require separate assembly and reference arrays alongside the compute kernel.
